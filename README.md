# SQLearningBot
Telegram‑бот для изучения SQL с проверкой решений через AI API.
Поддерживает систему баллов, статистику, тренировочный режим,
историю решений, экспорт в CSV и подсказки.

---

## О проекте
Данный бот разработан как интерактивный тренажёр
для практического освоения языка SQL.

Весь учебный контент (задачи, упражнения, теоретические материалы)
основан на методическом пособии «PostgreSQL с нуля».
Без этого пособия бот теряет свою образовательную ценность,
так как все задания и объяснения берутся непосредственно из него.

Само пособие разработано на основе работы Касимовской О.А.
"Обучение проектированию и разработке баз данных в MySQL
в рамках курса по выбору для технологического профиля",
посвященной MySQL, но полностью адаптировано под PostgreSQL.

Пользователь может получить PDF-версию пособия
в боте по команде `/materials`.

**Дополнительно** в боте реализована секция **«Повышенный уровень» (ADVANCED)**.
Она предназначена для тех, кто уже полностью прошёл методическое пособие.
В этой секции представлены усложнённые задачи.
К каждой задаче даётся краткая теоретическая вставка.

## Требования
Для запуска нужен **Docker** и **Docker Compose**

---

## Запуск
### 1. Склонируйте репозиторий:
```bash
git clone https://github.com/quperino/SQLearningBot.git
cd SQLearningBot
```

### 2. Настройте переменные окружения
Скопируйте пример и отредактируйте файл, указав реальные значения:
```bash
cp .env.example .env
```

### 3. Запустите все сервисы (PostgreSQL, RabbitMQ, сам бот):
```bash
docker compose up -d
```

---

## Добавление новых задач
Все задачи хранятся в SQL-скриптах миграций (папка `app/src/main/resources/db/migration`). Чтобы добавить новые задачи, создайте новый файл миграции (например, `V4__add_new_tasks.sql`) с идемпотентными `INSERT`-ами (используйте `WHERE NOT EXISTS` или `ON CONFLICT`).

После создания файла миграции пересоберите Docker-образ:
```bash
docker compose up --build
```

## Непрерывная интеграция и доставка (CI/CD)
В репозитории настроены автоматические пайплайны для **GitHub Actions**.
- **GitHub Actions** выполняет полный цикл:
  - Запуск всех тестов (`mvn clean test`).
  - Сборка JAR-файла.
  - Сборка Docker-образа.
  - Публикация образа на Docker Hub (`qeperino/sqlearningbot:latest`) при пуше в ветку `main`.
- **GitFlic** и **GitVerse** используются только как **зеркало репозитория** без CI/CD.

## Тестирование
Для тестов используется отдельная база H2 и файл `db/data-test.sql`. Тесты **не требуют** запущенных контейнеров Docker.

## Команды бота
- `/start` – начать работу (запросит email при первом запуске)
- `/help` – показать список команд
- `/cancel` – отменить текущее действие (например, решение задачи)
- `/reset` – полный сброс прогресса (баллы, решения, email)
- `/stats` – статистика: баллы, решённые задачи по секциям
- `/history` – история решений с пагинацией (статус, дата, попытки)
- `/export` – экспорт истории решений в CSV (только в состоянии ожидания)
- `/hint` – подсказка по текущей задаче (также доступна как кнопка)
- `/materials` – получить методическое пособие (PDF)

## Особенности реализации
- **Проверка решений** – через внешний AI API (совместимый с OpenAI). Тайм-аут – 1 минута.
- **Система баллов** – начисляются за верно решённые задачи (защита от повторного начисления).
- **Тренировочный режим** – повторное решение уже пройденных задач без начисления баллов.
- **Сброс решения** – позволяет «очистить» прогресс по задаче (баллы вычитаются, задачу можно решить заново).
- **История решений** – пагинация по 5 записей.
- **Экспорт в CSV** – кодировка UTF‑8 с BOM для корректного отображения в Excel.
- **Подсказки** – хранятся в таблице tasks, отмечаются как использованные.
- **Логирование времени обработки** – сохраняется в таблицу bot_messages.
- **Валидация JSON** – все сообщения из очередей RabbitMQ проверяются на синтаксическую корректность.

## Управление опасными функциями
В файле `app/src/main/resources/application.properties` можно отключить
команду полного сброса пользователя (`/reset`)
и кнопку сброса решения в тренировочном режиме.
Это полезно для production-окружения, чтобы предотвратить
случайную или намеренную потерю прогресса пользователей.
```properties
# Отключить команду /reset (по умолчанию true)
app.features.reset-command.enabled=false
```
```properties
# Отключить кнопку "Сбросить решение" (по умолчанию true)
app.features.task-reset-button.enabled=false
```
